# 维护协议

## 1. 语义锚点允许列表维护

- **责任人**：文档维护者（Maintainer）
- **触发条件**：新增动态/生成/模板化锚点，或现有 Tier 2 锚点失效
- **操作**：在 `.anchor-allowlist.md` 追加条目，填写 `reviewed_by` 与 `review_timestamp`（ISO8601）

## 2. 结构一致性规则版本化

- Gate 3.5 的四个不变量（接口矩阵、状态所有权、签名字符串、交叉引用可见性）以脚本代码为唯一权威
- 任何不变量变更须通过 PR 修改 `scripts/gate_035_structure.py`

## 3. 人工语义矛盾审阅周期

- **触发**：每次 `03-system-design.md` 或 `04-functional-design.md` 发生实质性变更（修改了接口签名、状态变量定义、错误码含义或行为描述）
- **记录**：审阅完成后追加 `Signed-off-by: <name> <email> <ISO8601_date>`
- **当前签署**：（留空，供 Phase 6 填写）

```text
Signed-off-by:
```

## 4. 门禁脚本升级流程

- 扫描器变更须保证向后兼容
- 校验器变更须配套单元测试

## 5. 边界矩阵变更控制

- 6 个关注点的 Canonical/Ref/Expand 关系由 `DOC_CONTRACT.md` 定义
- 若需将"局部展开"列从 `N` 改为 `Y`，须经 Architect 审批
